##  Branching Rules

* Always create branches from **`dev`**
*  Do NOT create branches from `main`
* Branch naming format:

```
<your-name>-dev
```

**Example:**

```
tusar-dev
manthan-dev
```

##  Commit Standard
 Verified Commits (Mandatory)

All commits must be Verified (*** signed).
Why Verified Commits are Important
Confirms the commit was made by the real author
Prevents identity spoofing
Improves security and trust

Follows professional and enterprise standards

### PR Title:

Clear and meaningful.

* What was implemented?
